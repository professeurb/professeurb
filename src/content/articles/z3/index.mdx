Nous allons résoudre ce problème en deux étapes. Dans un premier temps, à l'aide d'un solveur de [_satisfiabilité modulo des théories_](https://fr.wikipedia.org/wiki/Satisfiability_modulo_theories), nous allons traduire une partie des contraintes sous forme logique et utiliser le solveur pour déterminer les solutions compatibles avec ces contraintes. ## Présentation de Z3

Notre outil principal va être le solveur [Z3](https://github.com/Z3Prover/z3) développé par Microsoft, qui permet de vérifier si des contraintes logiques et arithmétiques peuvent être satisfaites et donner, le cas échéant, des valeurs aux variables qui satisfont les contraintes.

Pour l'utiliser en Python, il vous faudra probablement l'installer (avec `py§pip` ou de façon plus manuelle comme expliqué sur la page de présentation du solveur).

Voici un exemple très simple d'utilisation. Commençons par importer le module et par déclarer quelques variables entières.

```python
>>> from z3 import *
>>> x = Int('x')
>>> y = Int('y')
>>> z = Int('z')
```

Nous allons ensuite définir un solveur auquel nous allons ajouter un certain nombre de contraintes&nbsp;:

```python
>>> s = Solver()
>>> s.add(x*x + y*y == z*z)
>>> s.add(z <= 10)
>>> s.add(z >= 0)
>>> s.add(x >= 1)
>>> s.add(y >= 1)
```

Ici, nous avons donc comme contraintes que

$$
x^2 + y^2 = z^2, \qquad 0 \leq z \leq 10, \qquad 1 \leq x \qquad \mathrm{et} \qquad 1 \leq y
$$

Est-il possible de satisfaire ces contraintes&nbsp;?

```python
>>> s.check()
sat
```

La réponse `py§sat` indique que c'est le cas, `py§s` a réussi à déterminer des valeurs qui conviennent. Pour récupérer ces valeurs, on procède ainsi&nbsp;:

```python
>>> s.model()
[z = 10, y = 8, x = 6]
```

Le modèle proposé peut s'utiliser comme un dictionnaire&nbsp;:

```python
>>> m = s.model()
>>> m[x]
6
```

Par contre, ici, `py§m[x]` n'est pas vraiment un entier&nbsp;:

```python
>>> type(m[x])
<class 'z3.z3.IntNumRef'>
```

et on ne peut pas directement tester sa valeur&nbsp;:

```python
>>> m[x] == 2
6 == 2
>>> type(m[x] == 2)
<class 'z3.z3.BoolRef'>
```

Pour cela, il faut utiliser `py§as_long()` pour le convertir en véritable entier&nbsp;:

```python
>>> m[x].as_long()
6
>>> type(m[x].as_long())
<class 'int'>
>>> m[x].as_long() == 2
False
```

Pour plus de renseignements, je vous conseille l'excellente [page](https://ericpony.github.io/z3py-tutorial/guide-examples.htm) d'introduction à Z3 en Python d'[Eric Pony](https://github.com/ericpony/z3py-tutorial).

## Modélisation du problème

Nous allons tout d'abord créer une variable pour chaque case de la grille&nbsp;:

```python
C = [[Int("c{}{}".format(l, c)) for c in range(7)] for l in range(7)]
```

Avec ces variables, nous allons coder les différentes contraintes du puzzle à l'aide d'une fonction `py§probleme` prenant `py§C` comme argument&nbsp;:

```python
def probleme(C):
    contraintes = []
    ...
```

1. Chaque case non vide contient un nombre compris entre `py§1` et `py§9`. Il est très pratique de supposer de plus qu'une case vide contient le nombre&nbsp;`0`.

```python
    ...
    # encadrement des valeurs des cellules
    for l in range(7):
        for c in range(7):
            contraintes.append(And(C[l][c] >= 0, C[l][c] <= 9))
    ...
```

2. La somme de chaque ligne et de chaque colonne est égale à `py§20`. On utilise la fonction `py§Sum` de `py§z3` pour simplifier l'écriture de la contrainte.

```python
    ...
    # sommes des lignes
    for l in range(7):
        contraintes.append(Sum([C[l][c] for c in range(7)]) == 20)
    # sommes des colonnes
    for c in range(7):
        contraintes.append(Sum([C[l][c] for l in range(7)]) == 20)
    ...
```

3. Chaque carré `py§2 x 2` contient au moins une case vide&nbsp;:

```python
    ...
    # les carrés 2x2 ne sont pas entièrement pleins
    for l in range(6):
        for c in range(6):
            contraintes.append(Or(
                C[l][c] == 0,
                C[l + 1][c] == 0,
                C[l][c + 1] == 0,
                C[l + 1][c + 1] == 0
            ))
    ...
```

4. Chaque ligne et chaque colonne contient exactement 4 valeurs non nulles, et donc 3 valeurs nulles. Pour traîter cette contrainte, nous avons besoin d'énumérer les ensembles de 3 entiers compris entre `py§0` et `py§6`. Pour cela, nous allons utiliser la bibliothèque [`py§itertools`](https://docs.python.org/3/library/itertools.html) qui permet de faire cela très simplement&nbsp;:

```python
>>> import itertools
>>> pos = itertools.combinations(range(7), 3)
>>> list(pos)
[(0, 1, 2), (0, 1, 3), (0, 1, 4), (0, 1, 5), (0, 1, 6), (0, 2, 3),
 (0, 2, 4), (0, 2, 5), (0, 2, 6), (0, 3, 4), (0, 3, 5), (0, 3, 6),
 (0, 4, 5), (0, 4, 6), (0, 5, 6), (1, 2, 3), (1, 2, 4), (1, 2, 5),
 (1, 2, 6), (1, 3, 4), (1, 3, 5), (1, 3, 6), (1, 4, 5), (1, 4, 6),
 (1, 5, 6), (2, 3, 4), (2, 3, 5), (2, 3, 6), (2, 4, 5), (2, 4, 6),
 (2, 5, 6), (3, 4, 5), (3, 4, 6), (3, 5, 6), (4, 5, 6)]
```

Cela nous permet de traduire facilement la contrainte qu'exactement 3 cases par ligne et par colonne sont vides. Notons l'utilisation de la construction `py§... if ... else ...` pour pouvoir faire un test utilisé comme une expression (ici, à l'intérieur d'une liste en compréhension) (Dans de nombreux langages, on note cela en utilisant l'opérateur ternaire `cpp§test ? si_vrai : si_faux` alors qu'en Ocaml et plus généralement avec des langages fonctionnels, un `ml§if` classique convient car il n'y a pas de distinction entre expression et instruction).

```python
    ...
    # 3 cases vides par ligne
    for l in range(7):
        contraintes.append(Or([
            And([
                C[l][c] == 0 if c in positions else C[l][c] != 0
                for c in range(7)
            ])
            for positions in itertools.combinations(range(7), 3)
        ]))
    # 3 cases vides par colonne
    for c in range(7):
        contraintes.append(Or([
            And([
                C[l][c] == 0 if l in positions else C[l][c] != 0
                for l in range(7)
            ])
            for positions in itertools.combinations(range(7), 3)
        ]))
    ...
```

5. Reste à prendre en compte les données initiales de la grille. Pour cela, nous allons la traduire sous forme d'une chaîne de caractères&nbsp;:

```python
    ...
    # valeurs initiales de la grille
    grille = [
        ".4.....",
        "..63..6",
        ".....55",
        "...4...",
        "47.....",
        "2..74..",
        ".....1."
    ]
    for l in range(7):
        for c in range(7):
            valeur = grille[l][c]
            if valeur != ".":
                contraintes.append(C[l][c] == int(valeur))
    ...
```

En résumé, on a la fonction suivante&nbsp;:

```python
def probleme(C):
    s = Solver()
    # encadrement des valeurs des cellules
    for l in range(7):
        for c in range(7):
            s.add(And(C[l][c] >= 0, C[l][c] <= 7))
    # sommes des lignes
    for l in range(7):
        s.add(Sum([C[l][c] for c in range(7)]) == 20)
    # sommes des colonnes
    for c in range(7):
        s.add(Sum([C[l][c] for l in range(7)]) == 20)
    # les carrés 2x2 ne sont pas entièrement pleins
    for l in range(6):
        for c in range(6):
            s.add(Or(
                C[l][c] == 0,
                C[l + 1][c] == 0,
                C[l][c + 1] == 0,
                C[l + 1][c + 1] == 0
            ))
    # 3 cases vides par ligne
    for l in range(7):
        s.add(Or([
            And([
                C[l][c] == 0 if c in positions else C[l][c] != 0
                for c in range(7)
            ])
            for positions in itertools.combinations(range(7), 3)
        ]))
    # 3 cases vides par colonne
    for c in range(7):
        s.add(Or([
            And([
                C[l][c] == 0 if l in positions else C[l][c] != 0
                for l in range(7)
            ])
            for positions in itertools.combinations(range(7), 3)
        ]))
    # valeurs initiales de la grille
    grille = [
        ".4.....",
        "..63..6",
        ".....55",
        "...4...",
        "47.....",
        "2..74..",
        ".....1."
    ]
    for l in range(7):
        for c in range(7):
            valeur = grille[l][c]
            if valeur != ".":
                s.add(C[l][c] == int(valeur))
    # on a fini
    return s
```

## Toutes les solutions

Si l'on redemande un modèle, même après un nouveau `py§check()`, on obtient le même modèle.

```python
>>> s.check()
sat
>>> s.model()
[c23 = 0,
 c43 = 0,
 ...
 c42 = 2]
```

La raison est que le solveur va utiliser le même algorithme. Et comme la situation n'a pas changée… le résultat est lui aussi inchangé. Il faut donc ajouter une nouvelle contrainte pour être sûr de ne pas retomber sur le résultat précédent. Mais un nouveau modèle signifie avoir des nouvelles valeurs pour les variables `py§C[l][c]`. Plus précisément, il faut qu'au moins une valeur diffère. La contrainte est donc

$$
c_{0,0 } \neq m_{0, 0} \vee c_{0, 1} \neq m_{0, 1} \vee \ \cdots \ \vee c_{6, 6} \neq m_{6, 6}
$$

où «&nbsp;$\vee$&nbsp;» désigne la disjonction, $c_{i,j}$ la variable correspondant de la case $(i,j)$ et $m_{i,j}$ la valeur donnée par le modèle. Cela se traduit par&nbsp;:

```python
>>> s.add(Or([C[l][c] != m[C[l][c]] for l in range(7) for c in range(7)]))
```

On ne peut donc obtenir à nouveau le modèle précédent. Vérifions cela&nbsp;:

```python
>>> s.check()
sat
>>> affiche_modele(s.model())
7 4 . . 7 . 2
. 5 6 3 . . 6
. 4 . . 6 5 5
7 . 2 4 . 7 .
4 7 6 . 3 . .
2 . . 7 4 7 .
. . 6 6 . 1 7
```

Il faut donc répéter ce processus jusqu'à ce que les contraintes ne puissent plus être satisfaites pour obtenir toutes les solutions. On en déduit la fonction suivante&nbsp;:

```python
def solutions_temporaires():
    sols = []
    s = probleme(C)
    while s.check() == sat:
        m = s.model()
        sols.append(m)
        # on interdit de retrouver la solution actuelle
        s.add(Or([C[l][c] != m[C[l][c]] for l in range(7) for c in range(7)]))
    return sols
```

Si l'on a maintenant obtenu toutes les solutions, il y en a encore trop, puisque certaines contraintes restent à prendre en compte&nbsp;:

```python
>>> sols = solutions_temporaires()
>>> len(sols)
1140
```

Une des contraintes restantes est très simple à mettre en place, celle concernant le nombre d'occurences de chaque valeur.

```python
def teste_occurences(m):
    vals = [m[C[l][c]].as_long() for l in range(7) for c in range(7)]
    for v in range(1, 8):
        if vals.count(v) != v:
            return False
    return True
```

On en déduit une nouvelle fonction de détermination des solutions&nbsp;:

```python
def solutions_temporaires2():
    sols = []
    s = probleme(C)
    while s.check() == sat:
        m = s.model()
        if teste_occurences(m):
            sols.append(m)
        # on interdit de retrouver la solution actuelle
        s.add(Or([C[l][c] != m[C[l][c]] for l in range(7) for c in range(7)]))
    return sols
```

```python

```

## Vérification de la connexité

Pour traduire les contraintes de connexité (les cases numérotées doivent être connexes, en un seul morceau), nous allons utiliser l'algorithme [Union-Find](https://fr.wikipedia.org/wiki/Union-find).

### Présentation de l'algorithme

L'idée est représenter une partition d'ensembles d'entiers à l'aide d'un tableau.

Supposons que l'ensemble des entiers de $0$ à $9$ soit partitionné ainsi&nbsp;:

$$
\{ 0, 4 \}, \{ 2 \}, \{ 3, 6, 8, 9 \}, \{ 1, 5, 7 \}
$$

Une représentation possible de cette partition est le tableau `py§t` suivant&nbsp;:

```python
>>> t = [  4,  7, -1,  8, -2, -3, -4,  5,  6,  6]
>>> for i in range(len(t)):
...     print("t[{}] = {:2}".format(i,t[i]), end="\n" if i % 5 == 4 else "   ")
...
t[0] =  4   t[1] =  7   t[2] = -1   t[3] =  8   t[4] = -2
t[5] = -3   t[6] = -4   t[7] =  5   t[8] =  6   t[9] =  6
```

Remarquons tout d'abord que dans le tableau `py§t`, seuls 4 indices ont une valeur strictement négatives&nbsp;:

```python
>>> [i for i in range(len(t)) if t[i] < 0]
[2, 4, 5, 6]
```

Il y en a exactement un par sous-ensemble de la partition. C'est le **représentant** du sous-ensemble. Par exemple, $\{ 3, 6, 8, 9\}$ a `py§6` pour représentant. La valeur associée dans `py§t` à chacun de ces représentants est l'opposé du nombre d'éléments du sous-ensemble correspondant. Ainsi, `py§t[5] = -3` dont le sous-ensemble représenté par `py§5` comporte 3 éléments.

L'interprétation des autres valeurs présentantes dans `py§t` est la suivante&nbsp;: en notant `py§y = t[x]`, si `py§y >= 0` cela signifie que `py§x` et `py§y` sont dans le même sous-ensemble de la partition, et en répétant le processus avec `py§y`, on arrivera au représentant. <!-- Par exemple, avec le tableau précédent, on a
`t[3] = 8`, `py§t[8] = 6` et `py§t[6] = -4`. On a donc un chemin partant de `py§3` qui arrive à `py§6` en passant par `py§8`. Ainsi, `py§3` est dans le sous-ensemble représenté par `py§6`. -->

```python
>>> i = 3
>>> while True:
...     print(i, end=" ")
...     if t[i] < 0:
...         print("")
...         break
...     print("->", end=" ")
...     i = t[i]
...
3 -> 8 -> 6
```

On en déduit la fonction `py§find` qui permet de trouver le représentant du sous-ensemble contenant un élément donné&nbsp;:

```python
def find(t, x):
    while t[x] >= 0:
        x = t[x]
    return x
```

Reste à voir comment se construit un tel tableau. Pour cela, l'autre fonction indispensable est `py§union` qui permet de fusionner les sous-ensembles dont on a donné deux éléments. Pour cela, on calcule les représentants de chacun des sous-ensembles, et on modifie le tableau en faisant pointer l'un vers l'autre. La taille permet de faire pointer le représentant du plus petit sous-ensemble vers l'autre, afin d'éviter d'avoir des chemins trop longs lors de la recherche des représentants.

Le code de la fonction s'en déduit directement.

```python
def union(t, a, b):
    x = find(t, a)
    y = find(t, b)
    if t[x] < t[y]:
        # y va pointer vers x
        t[x] += t[y]
        t[y] = x
    elif x != y:
        # il faut s'assurer que l'on fusionne
        # deux sous-ensembles distincts
        t[y] += t[x]
        t[x] = y
```

Notons pour finir qu'au départ, le tableau `py§t` ne contient que des valeurs `py§-1`, chaque élément constituant un sous-ensemble à lui tout seul.

### Application au puzzle

Nous allons maintenant créer une fonction qui vérifie que toutes les cases numérotées sont connexes. Pour cela, on applique l'algorithme Union-Find en fusionnant les cases adjacentes non vides et, à la fin, on vérifie que la composente d'une des cases non vide est de taille 28, le nombre de cases non vides.

```python
def numero_case(l, c):
    return 7 * l + c

def teste_connexite(m):
    vals = [[m[C[l][c]].as_long() for c in range(7)] for l in range(7)]
    t = [-1] * 7 * 7
    # on fusionne les cases remplies adjacentes sur une même ligne
    for l in range(7):
        for c in range(6):
            if (vals[l][c] != 0 and vals[l][c + 1] != 0):
                union(t, numero_case(l, c), numero_case(l, c + 1))
    # on fait de même verticalement
    for l in range(6):
        for c in range(7):
            if (vals[l][c] != 0 and vals[l + 1][c] != 0):
                union(t, numero_case(l, c), numero_case(l + 1, c))
    # on cherche une case non vide,
    # on sait qu'il y en a une sur la première ligne
    for c in range(7):
        if vals[0][c] > 0:
            break
    # on vérifie si sa composante connexe est de taille 28
    return t[find(t, c)] == -28
```

Reste à calculer le score. On rappelle qu'il s'agit du produit des aires des composantes connexes formées de cases vides. On adapte la fonction `py§teste_connexite`.

```python
def score(m):
    vals = [[m[C[l][c]].as_long() for c in range(7)] for l in range(7)]
    t = [-1] * 7 * 7
    # on fusionne les cases vides adjacentes sur une même ligne
    for l in range(7):
        for c in range(6):
            if (vals[l][c] == 0 and vals[l][c + 1] == 0):
                union(t, numero_case(l, c), numero_case(l, c + 1))
    # on fait de même verticalement
    for l in range(6):
        for c in range(7):
            if (vals[l][c] == 0 and vals[l + 1][c] == 0):
                union(t, numero_case(l, c), numero_case(l + 1, c))
    # on calcule le score, sachant que les cases remplies forment
    # chacune un sous-ensemble de taille 1, ce qui ne va pas
    # poser de problème pour le score.
    score = 1
    for v in t:
        if v < 0:
            score *= -v
    return score
```
